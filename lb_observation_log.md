## Next Public-Observation Experiments

1. Structural block target representation: raw overnight context can recover strict pseudo-hidden block state, but target-rate smoothing, raw posterior averaging, simple E56 distillation, within-block joint labels, transition residual hidden-sign moves, and transition-gated E56 teacher movement cannot align it at selector-scale margin. The next JEPA-style target should encode block topology/transition/label-world constraints while preserving row calibration and S3.
2. Independent non-anchor E56 validation: before another E56 teacher submission, find a train-only or pseudo-hidden stress that predicts which E56 cells are valid. A public improvement from a sub-margin distillation file would be too hard to interpret without this.
3. Target sensitivity probe: mixmin delta를 target groups별로 분리해 which target movement explains the new public order를 추정한다.
4. Next candidate generation should be mixmin-relative, not a2c8-relative: preserve the anchor-loss/binary-world direction while reducing private risk and identifying whether inverse7/raw-structure or subject-calendar block context can be blended without losing the new public gain.
5. Lower-risk S4+Q3 probes such as `analysis_outputs/submission_label_flow_focused_1bbfb735.csv` are now secondary diagnostics, not the next default public test. They should be revisited only after mixmin-relative selector calibration.
6. Public LB inverse fitting and simple subject/order/date/raw-domain masks remain underidentified. E48 adds a strong new anchor, but it does not by itself make inverse-LB optimization safe.
7. Block-state oracle evidence still explains why 0.54 feels far despite this improvement: the hidden rate vector exists, but current observable proxies do not identify it. Public submissions should not be expected to jump unless they change block-state inference or explicitly test a newly calibrated sensor worldview.
8. Current block-summary Ridge context is not that change. E47 moves only `0.014083` of the oracle gap and worsens direct block-rate loss, so it should be treated as a negative local sensor rather than a candidate source.
